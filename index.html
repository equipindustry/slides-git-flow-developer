Conceptos de Git

  • Sistema de control de versiones creado por Linus Torvalds.

Sistema de control de Versiones

  • Sistema que registra los cambios realizados sobre un archivo o conjunto de archivos a lo largo del tiempo.

Branch

  • Useful for 'book ended' actions
  • Simple exception handling
  • Built-in for File objects, etc. in 2.7

Hooks de Git

  • pre-commit
  • post-commit
  • prepare-commit-msg
  • post-receive

pre-commit

  • se ejecuta antes de realizar un commit

Uso:

  • Verificador de Codigo.
  • Ejecucion de Test Unit

post-commit

  • se ejecuta despues de realizar un commit

Uso:

  • versionado a archivos static usando un archivo last_commit.
  • notificaciones a usuarios email ...

prepare-commit-msg

  • se ejecuta antes de realizar el commit.

Uso:

  • agregar el nombre del branch al commit para gestor de proyectos como Jira, Redmine

post-receive

  • se ejecuta al realizar un push en el repositorio Bare.

Uso:

  • despliegues automaticos.

Flujo de Trabajo

  • El flujo de trabajo (workflow en inglés) es el estudio de los aspectos operacionales de una actividad de trabajo: cómo se estructuran las tareas, cómo se realizan, cuál es su orden correlativo, cómo se sincronizan, cómo fluye la información que soporta las tareas y cómo se le hace seguimiento al cumplimiento de las tareas. Wikipedia

Que es?

  • Un registro de cambios1 (también se usa la forma inglesa, changelog) es un archivo que lista los cambios hechos a un proyecto informático (como por ejemplo un sitio web o un proyecto de software) desde su última versión, incluyendo habitualmente corrección de errores malintencionados, nuevas características, etc. La mayoría de proyectos de código abierto incluyen un registro de cambios como uno de los ficheros más importantes de sus distribuciones. wikipedia
